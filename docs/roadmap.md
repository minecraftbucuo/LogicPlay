## Phase 6: 沉浸式音效体验

目标：为 LogicPlay 增加轻量、克制、可关闭的音效反馈，让玩家在点击、执行、通关、失败等关键操作中获得更明确的反馈，同时不打扰编程思考。

### Step 6.1: 按钮音效系统

- 状态：✅ 已完成第一版
- 目标：先为所有主要按钮增加统一点击音效，建立音效播放基础设施。
- 覆盖范围：
  - 主菜单：`开始游戏`、`关于`、`知道了`。
  - 关卡选择：进入关卡、返回主菜单。
  - 游戏界面：返回关卡选择、执行、快速执行、停止、重置关卡、提示说明、显示标准答案、放大编辑、弹窗关闭。
  - 通关弹窗：关闭、返回关卡选择、进入下一关。
- 音效类型：
  - `ui-click`：普通按钮点击，短促、干净。
  - `ui-confirm`：开始游戏、进入关卡、完成编辑等确认操作，略微明亮。
  - `ui-back`：返回、关闭弹窗，音色更轻。
  - `ui-stop`：停止执行，使用低一点的提示音，和普通点击区分。
- 技术方案：
  - 新增 `src/audio/AudioManager.ts` 管理音效播放。
  - 第一版使用 Web Audio API 合成短促 UI 音效，不依赖外部素材文件。
  - 后续如果需要更强风格化，可以替换为 `public/audio/` 下的素材文件。
  - 所有播放失败都静默处理，不能影响游戏操作。
- 资源规划：
  - 第一版无需音频素材文件，直接用 Web Audio API 生成按钮反馈音。
  - 如果后续改用素材文件，统一放在 `public/audio/`。
  - 素材版建议文件名：`ui-click.mp3`、`ui-confirm.mp3`、`ui-back.mp3`、`ui-stop.mp3`。
  - 音量保持低调，默认按钮音效音量建议 `0.35`。
- 验收标准：
  - 点击主要按钮能听到反馈。
  - 快速连续点击不会报错、不会明显卡顿。
  - 浏览器自动播放限制下，第一次用户点击后音效可正常播放。
  - 静音或播放失败时不影响按钮原本功能。

### Step 6.2: 音效开关与音量设置

- 状态：✅ 已完成
- 目标：给玩家控制权，避免音效打扰。
- 功能：
  - 全局音效控制面板（静音按钮 + 音量滑块），固定在右上角，所有页面可见。
  - 设置保存到 `localStorage`，刷新页面后保持。
  - 默认开启音效，主音量 0.5。
  - 静音时所有音效不播放；调节音量时实时生效。
  - 拖动音量滑块时自动取消静音。
- 技术方案：
  - `AudioManager` 增加 `muted`/`volume` 状态和 `setMuted()`/`setVolume()` 方法。
  - `play()` 中静音直接返回，音量乘以主音量系数。
  - 新增 `AudioSettings` 组件，集成到 `App.tsx` 顶层，所有页面共享。
  - localStorage key: `logicplay-audio-settings`。
- 验收标准：
  - 关闭音效后所有按钮音效停止播放。
  - 刷新页面后保持上次设置。
  - 音量滑块实时控制音效大小。

### Step 6.3: 游戏事件音效

- 状态：✅ 已完成
- 目标：让游戏反馈更有层次，但不抢占按钮音效第一阶段。
- 事件：
  - `robot-step`：机器人移动一步（慢速模式）。
  - `robot-turn`：机器人转向（慢速模式）。
  - `collision`：撞墙失败。
  - `collect`：收集星星。
  - `success`：通关成功。
  - `test-pass`：多测试用例单项通过。
  - `test-fail`：多测试用例失败（预留，暂未使用）。
- 技术方案：
  - `AudioManager` 扩展 `GameSoundType` 和 `SoundType`，新增 7 种游戏事件音效的合成参数。
  - 部分音效使用非 sine 波形（`collision` 用 sawtooth，`test-fail` 用 square）增加辨识度。
  - `GamePlay` 执行循环中在关键节点调用 `audioManager.play()`。
  - 快速执行模式（20ms/步）跳过 `robot-step` 和 `robot-turn`，只播放关键事件音效，避免变成噪音。
- 验收标准：
  - 慢速执行时能听到每步移动/转向音效。
  - 碰撞、收集、通关有明确音效反馈。
  - 快速执行不会产生噪音。

### Step 6.4: 背景音乐

- 状态：✅ 已完成
- 目标：为游戏添加背景音乐，增强沉浸感。
- 功能：
  - 全局统一背景音乐（`/audio/bgm.mp3`），所有界面共用。
  - BGM 独立开关和音量控制，默认音量较低（0.15）。
  - 通关音效覆盖在 BGM 之上，不冲突。
- 技术方案：
  - `AudioManager` 用 `HTMLAudioElement` 加载外部 mp3 文件，支持循环播放。
  - BGM 在用户首次点击按钮时启动（浏览器自动播放策略要求用户手势）。
  - BGM 音量独立于音效音量，持久化到 localStorage。
  - `AudioSettings` 组件增加可展开的 BGM 控制面板。
  - mp3 文件放在 `public/audio/` 目录下。
- 音乐来源：
  - 从 Pixabay（CC0 协议，可商用）下载。
  - 手动下载后放入 `public/audio/bgm.mp3`。
- 注意：
  - 背景音乐有明确开关，默认开启但音量低。
  - 静音时自动停止 BGM，取消静音后恢复。
  - 优先保证音效体验，BGM 不干扰玩家读代码和思考。

### Phase 6 实施顺序

1. 先实现 `AudioManager` 和按钮点击音效。
2. 给所有主要按钮接入合适的音效类型。
3. 增加音效开关和本地持久化。
4. 再接入游戏事件音效。
5. 最后评估是否加入背景音乐。
