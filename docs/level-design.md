## Level 1：基础移动

- 状态：✅ 已实现
- 关卡目标：让玩家学习最基础的移动。
- 教学重点：使用 `robot.move_forward()` 控制机器人向前移动。
- 通关目标：移动到终点即可通关，并解锁下一关。
- 地图尺寸：5x5。
- 起点：`{ x: 0, y: 2, direction: 'right' }`
- 终点：`{ x: 4, y: 2 }`
- 障碍物：无。
- 收集品：无。
- 特殊机制：进入第一关时先显示一段故事背景，居中展示在屏幕中间；玩家点击确认后再开始游戏。
- 通关条件：机器人到达终点格。
- 初始代码：只提供最基础的一行移动提示，不给出完整答案。
- 标准答案：通过游戏界面“显示标准答案”按钮查看。
- 提示说明：通过游戏界面“提示说明”按钮查看过关思路和当前关卡命令/API 用法。

```python
# 提示：机器人需要多次向前移动
robot.move_forward()
# 继续补充代码，让机器人到达终点
```

- 设计说明：这一关只让玩家建立“写一行代码，机器人移动一步”的直觉，不引入转向、障碍物或其他机制。

- 地图尺寸：12x12。
- 起点：`{ x: 0, y: 6, direction: 'right' }`

- 障碍物：无。
- 收集品：无。

- 标准答案：通过游戏界面“显示标准答案”按钮查看。
- 提示说明：通过游戏界面“提示说明”按钮查看过关思路和当前关卡命令/API 用法。


## Level 4：判断前方

- 状态：✅ 已实现
- 关卡目标：让玩家开始学习 `if` 条件分支语句。
- 教学重点：根据 `robot.sense()` 的结果判断前方是否可走，而不是写死固定路线。
- 新增 API：`robot.sense()`。
- API 设计：`robot.sense()` 返回机器人前方一格的内容，例如 `wall`、`empty`、`target`。
- 新增要素：`?` 神秘方块。
- 神秘方块规则：它看起来永远是 `?`，但实际可能是墙，也可能是空气。
- 随机规则：第四关中一整列都是 `?` 神秘方块，每次进入或重置关卡时，随机且仅随机一个神秘方块是空气，其余全部是墙。
- 通关目标：机器人必须用条件判断找到唯一通路，穿过神秘方块列并到达终点。
- 地图尺寸：7x7。
- 起点：`{ x: 0, y: 0, direction: 'right' }`
- 终点：`{ x: 6, y: 6 }`，保持固定，避免玩家通过终点位置反推空气格。
- 神秘方块列：`x = 3`，`y = 0..6`。
- 收集品：无。
- 特殊机制：进入第四关时先显示一段故事背景，居中展示在屏幕中间；玩家点击确认后再开始游戏。
- 通关条件：机器人穿过本局唯一空气神秘方块，并到达对应终点格。
- 初始代码：只提供 `if` 判断结构提示，不给出完整扫描神秘列的答案。
- 标准答案：通过游戏界面“显示标准答案”按钮查看。
- 提示说明：通过游戏界面“提示说明”按钮查看过关思路、`robot.sense()` 返回值和当前关卡命令/API 用法。

```python
# 提示：先走到神秘方块列前
for i in range(2):
    robot.move_forward()

# 提示：用 if 判断前方是不是墙
if robot.sense() == 'wall':
    robot.turn_right()
else:
    robot.move_forward()
```

- 设计说明：这一关刻意防止玩家用固定路线通关。因为唯一空气格每次都会变，玩家只有使用 `robot.sense()` 和 `if` 判断，才能稳定找到通路。
