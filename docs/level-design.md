## Level 4：判断前方

- 状态：✅ 已实现
- 关卡目标：让玩家开始学习 `if` 条件分支语句。
- 教学重点：根据 `robot.sense()` 的结果判断前方是否可走，而不是写死固定路线。
- 新增 API：`robot.sense()`。
- API 设计：`robot.sense()` 返回机器人前方一格的内容，例如 `wall`、`empty`、`target`。
- 新增要素：`?` 神秘方块。
- 神秘方块规则：它看起来永远是 `?`，但实际可能是墙，也可能是空气。
- 随机规则：第四关中一整列都是 `?` 神秘方块，每次进入或重置关卡时，随机且仅随机一个神秘方块是空气，其余全部是墙。
- 通关目标：机器人必须用条件判断找到唯一通路，穿过神秘方块列并到达终点。
- 地图尺寸：7x7。
- 起点：`{ x: 0, y: 0, direction: 'right' }`
- 终点：`{ x: 6, y: 6 }`，保持固定，避免玩家通过终点位置反推空气格。
- 神秘方块列：`x = 3`，`y = 0..6`。
- 收集品：无。
- 特殊机制：进入第四关时先显示一段故事背景，居中展示在屏幕中间；玩家点击确认后再开始游戏。
- 通关条件：机器人穿过本局唯一空气神秘方块，并到达对应终点格。
- 初始代码：只提供 `if` 判断结构提示，不给出完整扫描神秘列的答案。
- 标准答案：通过游戏界面“显示标准答案”按钮查看。
- 提示说明：通过游戏界面“提示说明”按钮查看过关思路、`robot.sense()` 返回值和当前关卡命令/API 用法。

```python
# 提示：先走到神秘方块列前
for i in range(2):
    robot.move_forward()

# 提示：用 if 判断前方是不是墙
if robot.sense() == 'wall':
    robot.turn_right()
else:
    robot.move_forward()
```

- 设计说明：这一关刻意防止玩家用固定路线通关。因为唯一空气格每次都会变，玩家只有使用 `robot.sense()` 和 `if` 判断，才能稳定找到通路。

## Level 5：封装动作

- 状态：✅ 已实现
- 关卡目标：让玩家学会使用函数，把“绕过一组门并收集星星”的动作封装起来并多次调用。
- 教学重点：使用 Python 的 `def` 定义函数，用函数名加括号调用函数。
- 设计动机：前几关已经学习了移动、转向、循环和条件判断。这一关引入“把一组指令命名并复用”的思想，让玩家理解函数不是为了少写一行代码，而是为了让重复路线更清晰、更容易维护。
- 通关目标：机器人依次绕过三组相同结构的障碍门，收集每个门后的星星，并抵达终点。
- 地图尺寸：12x12。
- 起点：`{ x: 0, y: 8, direction: 'right' }`
- 终点：`{ x: 11, y: 8 }`
- 障碍物：三组相同的“门形障碍”，每组都会挡住直线路径，必须执行同一套绕行动作。
- 障碍布局：
  - 第一组门：`x = 2`，墙体位于 `{ x: 2, y: 7 }`、`{ x: 2, y: 8 }`、`{ x: 2, y: 9 }`。
  - 第二组门：`x = 5`，墙体位于 `{ x: 5, y: 7 }`、`{ x: 5, y: 8 }`、`{ x: 5, y: 9 }`。
  - 第三组门：`x = 8`，墙体位于 `{ x: 8, y: 7 }`、`{ x: 8, y: 8 }`、`{ x: 8, y: 9 }`。
- 可通行路线：每组门上方留出绕行通道，玩家需要重复执行“上绕、横穿、下回原线并收集中间墙后的星星”的动作。
- 收集品：三颗星星，分别位于三组墙体中间墙的正后方：`{ x: 3, y: 8 }`、`{ x: 6, y: 8 }`、`{ x: 9, y: 8 }`。
- 特殊机制：进入第五关时先显示一段故事背景，居中展示在屏幕中间；玩家点击确认后再开始游戏。
- 通关条件：机器人收集全部三颗星星，并到达终点格。
- 初始代码：只提供函数定义和调用结构提示，不给出完整绕行动作。
- 标准答案：通过游戏界面“显示标准答案”按钮查看。
- 提示说明：通过游戏界面“提示说明”按钮查看过关思路、函数定义/调用方式和当前关卡命令/API 用法。

```python
# 提示：把“绕过一组门并拿到星星”的动作封装成函数
def bypass_gate():
    robot.turn_left()
    # 在这里补完整绕行动作

# 每组门后都有星星，重复调用函数会更清晰
bypass_gate()
bypass_gate()
bypass_gate()
```

- 设计说明：这一关通过“每组门后都有必须收集的星星”来制造真实重复需求。玩家需要三次完成同一套“绕门并拿星星”的动作，函数不再是生硬限制，而是最自然的表达方式：把这套动作命名为 `bypass_gate()`，每遇到一组门就调用一次。
- 防止误解：文案里要明确“定义函数不会自动执行，必须调用函数才会执行函数体”。
- 进阶提示：函数体内部仍然可以使用已经学过的 `for` 循环，例如用循环写连续前进。
