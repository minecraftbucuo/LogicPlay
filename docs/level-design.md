## Level 4：判断前方

- 状态：✅ 已实现
- 关卡目标：让玩家开始学习 `if` 条件分支语句。
- 教学重点：根据 `robot.sense()` 的结果判断前方是否可走，而不是写死固定路线。
- 新增 API：`robot.sense()`。
- API 设计：`robot.sense()` 返回机器人前方一格的内容，例如 `wall`、`empty`、`target`。
- 新增要素：`?` 神秘方块。
- 神秘方块规则：它看起来永远是 `?`，但实际可能是墙，也可能是空气。
- 随机规则：第四关中一整列都是 `?` 神秘方块，每次进入或重置关卡时，随机且仅随机一个神秘方块是空气，其余全部是墙。
- 通关目标：机器人必须用条件判断找到唯一通路，穿过神秘方块列并到达终点。
- 地图尺寸：7x7。
- 起点：`{ x: 0, y: 0, direction: 'right' }`
- 终点：`{ x: 6, y: 6 }`，保持固定，避免玩家通过终点位置反推空气格。
- 神秘方块列：`x = 3`，`y = 0..6`。
- 收集品：无。
- 特殊机制：进入第四关时先显示一段故事背景，居中展示在屏幕中间；玩家点击确认后再开始游戏。
- 通关条件：机器人穿过本局唯一空气神秘方块，并到达对应终点格。
- 初始代码：只提供 `if` 判断结构提示，不给出完整扫描神秘列的答案。
- 标准答案：通过游戏界面“显示标准答案”按钮查看。
- 提示说明：通过游戏界面“提示说明”按钮查看过关思路、`robot.sense()` 返回值和当前关卡命令/API 用法。

```python
# 提示：先走到神秘方块列前

    robot.move_forward()
```

- 设计说明：这一关刻意防止玩家用固定路线通关。因为唯一空气格每次都会变，玩家只有使用 `robot.sense()` 和 `if` 判断，才能稳定找到通路。

## Level 6：迷宫策略

- 状态：✅ 已实现
- 章节定位：第二章开篇，从“语法学习”进入“策略与算法思维”。
- 关卡目标：让玩家写出能适应多张地图的通用迷宫策略，而不是为单张地图硬编码路线。
- 教学重点：综合使用 `while` 循环、`if/else` 条件判断、`robot.sense()` 感知和函数封装。
- 核心策略：贴墙走。推荐使用右手法则：优先尝试右转，如果右侧可走就右转并前进；否则如果前方可走就前进；否则左转。只要迷宫结构满足“墙体连通、终点在边界可达路径上”，贴墙策略就能到达终点。
- 通关目标：同一份代码依次通过多组迷宫测试用例，所有测试用例都到达终点才算通关。
- 地图尺寸：9x9。
- 起点/终点：每个测试用例可不同，但都保证起点方向与推荐贴墙策略匹配。
- 障碍物：墙体明显增多，形成单通道或少分支迷宫。
- 收集品：无。
- 特殊机制：多测试用例。玩家点击执行后，系统使用同一份代码依次加载每个测试用例，每个测试用例都从自己的起点、墙体和终点开始执行。
- 通关条件：所有测试用例均到达各自终点。
- 失败条件：任意测试用例撞墙、执行超出循环预算、指令执行完仍未到终点，则本关失败，并显示失败的是第几个测试用例。
- 初始代码：提供 `while` 循环和贴墙走函数结构，但不直接给完整判断逻辑。
- 标准答案：通过游戏界面“显示标准答案”按钮查看。
- 提示说明：通过游戏界面“提示说明”按钮查看贴墙走策略、`robot.sense()` 返回值、循环退出条件和多测试用例规则。

```python
# 提示：同一份代码会在多张迷宫里测试
# 可以把“右侧是否可走”的判断封装成函数

def follow_wall():
    # 在这里写贴墙走策略
    pass

while robot.sense() != 'target':
    follow_wall()
```

- 设计说明：这一关不应该只考“能不能写一串路线”，而是考“能不能写一个策略”。多测试用例会让固定路线失效；只要玩家写的是贴墙走策略，就能适应不同迷宫布局。
- 策略说明：为了支持右手法则，玩家需要判断右侧和前方。当前 API 只有 `robot.sense()` 能看前方，因此标准答案可以通过“临时右转感知，再转回原方向”的方式判断右侧：
  - 右转。
  - 如果前方不是 `wall`，说明原本右侧可走，直接前进。
  - 如果前方是 `wall`，左转回原方向，再判断前方是否可走。
  - 前方也不可走时，左转继续寻找方向。
- 循环退出：可以使用 `while robot.sense() != 'target':` 作为核心循环，但需要注意：当目标不在正前方而机器人已经站到目标格时，`sense()` 不会返回 `target`。因此实现时更推荐由引擎在每步后检测是否到达终点，玩家代码可以使用固定步数上限或 `for` 安全循环防止无限循环。
- 推荐标准答案思路：使用 `for step in range(200):` 包裹贴墙策略，避免玩家代码自己写死循环；引擎在机器人到达终点后判定当前测试用例通过。

## Level 7：随机猎手

- 状态：📝 设计中
- 章节定位：策略思维进阶，从"适应固定地图"到"应对随机变化"。
- 关卡目标：让玩家设计一种系统性搜索策略，保证无论星星随机出现在地图哪个位置，机器人都能找到并吃掉它。
- 教学重点：理解"穷举搜索"思想——用蛇形遍历（zigzag）等方式覆盖整个地图，确保不遗漏任何格子。综合使用 `while` 循环、`for` 循环、`if/else` 条件判断、`robot.sense()` 和函数封装。
- 核心策略：蛇形扫描。逐行遍历地图：沿当前方向前进直到撞墙，然后转向到下一行，反向前进，如此反复，像贪吃蛇一样覆盖整个地图。
- 通关目标：机器人收集 3 颗随机出现的星星。
- 地图尺寸：5x5。
- 起点：`{ x: 0, y: 0, direction: 'right' }`
- 障碍物：无（空旷地图，方便蛇形遍历）。
- 收集品：星星，共需收集 3 颗。
- 星星规则：
  - 地图上同时只有 1 颗星星。
  - 星星初始位置随机生成在一个空白格子上。
  - 机器人吃到星星后，下一颗星星立即随机出现在另一个空白格子上。
  - 星星不会出现在机器人占据的格子上。
  - 吃完 3 颗星星即通关。
- 特殊机制：
  - 随机性：每次进入或重置关卡时，星星位置都会重新随机生成。同一份代码必须能在任意随机种子下通关。
  - 多轮执行：玩家代码需要在吃完一颗星星后继续搜索下一颗，直到收集满 3 颗。
  - 步数上限：引擎设置最大执行步数（如 300 步），防止无限循环。
- 失败条件：撞墙、执行超出步数上限、或指令执行完仍未收集满 3 颗星星。
- 初始代码：只提供基本结构和提示，不给出完整搜索策略。
- 标准答案：通过游戏界面"显示标准答案"按钮查看。
- 提示说明：通过游戏界面"提示说明"按钮查看过关思路、蛇形扫描策略和当前关卡命令/API 用法。
