# LogicPlay 关卡设计

这个文件用于逐关设计正式关卡。当前按确认顺序逐关追加。

## Level 1：基础移动

- 状态：✅ 已实现
- 关卡目标：让玩家学习最基础的移动。
- 教学重点：使用 `robot.move_forward()` 控制机器人向前移动。
- 通关目标：移动到终点即可通关，并解锁下一关。
- 地图尺寸：5x5。
- 起点：`{ x: 0, y: 2, direction: 'right' }`
- 终点：`{ x: 4, y: 2 }`
- 障碍物：无。
- 收集品：无。
- 特殊机制：进入第一关时先显示一段故事背景，居中展示在屏幕中间；玩家点击确认后再开始游戏。
- 通关条件：机器人到达终点格。
- 初始代码：提供最基础的一行移动代码，让玩家理解调用方式。

```python
robot.move_forward()
```

- 设计说明：这一关只让玩家建立“写一行代码，机器人移动一步”的直觉，不引入转向、障碍物或其他机制。

### 第一关开场故事

- 展示时机：玩家进入第一关后，游戏正式开始前。
- 展示方式：屏幕中央弹窗，背景轻微变暗或模糊。
- 交互方式：玩家点击“开始任务”后关闭弹窗，再进入可操作状态。
- 故事文案：欢迎来到 LogicPlay。你的机器人已经在训练场待命，但它还不会自己行动。现在，请写下最基础的移动指令，让机器人一步一步抵达绿色终点。

## Level 2：绕过障碍

- 状态：✅ 已实现
- 关卡目标：让玩家学会通过转向绕过障碍，到达终点。
- 教学重点：使用 `robot.turn_left()` 和 `robot.turn_right()` 改变机器人朝向。
- 通关目标：绕过中间的障碍物，移动到绿色终点后通关并解锁下一关。
- 地图尺寸：5x5。
- 起点：`{ x: 0, y: 2, direction: 'right' }`
- 终点：`{ x: 4, y: 2 }`
- 障碍物：中间竖向墙体：`{ x: 2, y: 1 }`、`{ x: 2, y: 2 }`、`{ x: 2, y: 3 }`
- 收集品：无。
- 特殊机制：进入第二关时先显示一段故事背景，居中展示在屏幕中间；玩家点击确认后再开始游戏。
- 通关条件：机器人绕过障碍并到达终点格。
- 初始代码：提供转向命令示例，让玩家理解转向不会移动，只会改变朝向。

```python
# 绕过障碍物，到达绿色终点
robot.move_forward()
robot.turn_right()
robot.move_forward()
robot.turn_left()
```

- 设计说明：这一关承接第一关的直线移动，在地图中加入障碍物，迫使玩家使用左右转向规划路径。

### 第二关开场故事

- 展示方式：屏幕中央弹窗，背景轻微变暗或模糊。
- 交互方式：玩家点击“开始任务”后关闭弹窗，再进入可操作状态。
- 故事文案：训练场升起了第一道障碍墙。直线前进已经不够了，你需要让机器人学会转向：`robot.turn_left()` 可以向左转，`robot.turn_right()` 可以向右转。转向只会改变方向，不会前进。绕过障碍，抵达终点。

## Level 3：重复前进

- 状态：✅ 已实现
- 关卡目标：让玩家理解重复指令可以用 `for` 循环简化。
- 教学重点：使用 Python 的 `for` 循环重复执行 `robot.move_forward()`。
- 通关目标：在更大的训练场中移动到终点，通关后解锁下一关。
- 地图尺寸：12x12。
- 起点：`{ x: 0, y: 6, direction: 'right' }`
- 终点：`{ x: 11, y: 6 }`
- 障碍物：无。
- 收集品：无。
- 特殊机制：进入第三关时先显示一段故事背景，居中展示在屏幕中间；玩家点击确认后再开始游戏。
- 通关条件：机器人到达终点格。
- 初始代码：提供 `for` 循环结构，让玩家补充循环次数。

```python
# 使用循环重复前进
for i in range(5):
    robot.move_forward()
```

- 设计说明：这一关把网格扩大到 12x12，终点距离更远。如果继续逐行写 `robot.move_forward()` 会显得重复，因此自然引出 `for` 循环。

### 第三关开场故事

- 展示时机：玩家进入第三关后，游戏正式开始前。
- 展示方式：屏幕中央弹窗，背景轻微变暗或模糊。
- 交互方式：玩家点击“开始任务”后关闭弹窗，再进入可操作状态。
- 故事文案：训练场被扩展到了更远的距离。你当然可以一行一行写很多次 `robot.move_forward()`，但程序员会用更聪明的方式：`for` 循环可以让同一条指令重复执行。试着用循环，让机器人抵达远处的终点。
