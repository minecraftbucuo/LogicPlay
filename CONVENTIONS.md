# LogicPlay 开发规范

## 项目概述
LogicPlay - 一个边玩边学算法的 Web 游戏。玩家在 2D 地图上控制机器人，通过编写 Python 代码完成关卡。

## 技术选型
- **平台**: Web 网页
- **前端框架**: React
- **Python 执行**: Pyodide（浏览器端沙箱）
- **渲染**: Canvas 2D（像素/卡通风格）
- **玩家编程语言**: Python（需沙箱执行环境）
- **视觉风格**: 2D 像素/卡通，俯视角网格地图

## 核心开发原则

### 1. 小步迭代
- 一次只实现一个小功能
- 每次改动要小且可验证
- 不一次性生成大量代码

### 2. 先问后做
- 任何不明确的地方必须向用户提问，不要瞎猜
- 有多个方案时，列出选项让用户决定

### 3. 先说后做
- 写代码前，先告诉用户要做什么以及为什么
- 运行命令前，先告诉用户命令的作用和目的

### 4. 代码质量
- 优先编辑现有文件，而非创建新文件
- 不主动创建文档文件（除非用户要求）
- 不做超出需求范围的改动
- 不添加不必要的抽象和过度设计

### 5. 沟通规范
- 使用中文沟通
- 代码注释使用中文
- 保持简洁，直奔主题

## 算法教学路线（渐进式）
1. 基础语法：变量、循环、条件判断
2. 函数与模块化
3. 基础算法：搜索、排序
4. 进阶算法：路径搜索（BFS/DFS/A*）
5. 更多：动态规划、图算法等
